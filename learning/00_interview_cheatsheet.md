# ЧАСТЬ 1. ОПРЕДЕЛЕНИЯ

---

## Базовые понятия

| Термин | Что это (своими словами) |
|--------|--------------------------|
| **Переменная** | Бирка на предмете. Указывает на объект в памяти. Можно перевесить на другой предмет |
| **Функция** | Кусок кода с именем. Принимает аргументы, что-то делает, возвращает результат (или `None`) |
| **Метод** | Функция внутри класса. Первый аргумент всегда `self` — ссылка на объект |
| **Класс** | Чертёж для создания объектов. Данные (атрибуты) + поведение (методы) |
| **Объект** | Конкретная штука по чертежу. `user = User("Динислам")` — `User` = класс, `user` = объект |
| **Модуль** | Один `.py` файл. Можно импортировать из него что угодно |
| **Пакет** | Папка с `__init__.py`. Группирует модули |
| **Декоратор** | Функция-обёртка. `@pytest.fixture` — делает функцию фикстурой, не трогая её код |
| **Фикстура** | Подготовка данных для теста. Pytest сам подставляет по имени аргумента |
| **Контекстный менеджер** | `with` — гарантирует закрытие ресурса, даже если ошибка |

## Типы данных

| Тип | Описание | Мутабельный? | Пример |
|-----|----------|:------------:|--------|
| `int` | Целое число. Размер не ограничен | - | `42`, `-7` |
| `float` | Дробное число | - | `3.14` |
| `str` | Строка | - | `"hello"` |
| `bool` | Логический. **С большой буквы!** | - | `True`, `False` |
| `list` | Упорядоченный, изменяемый | **да** | `[1, 2, 3]` |
| `tuple` | Упорядоченный, НЕизменяемый | - | `(1, 2, 3)` |
| `dict` | Ключ-значение | **да** | `{"name": "Neo"}` |
| `set` | Уникальные, без порядка | **да** | `{1, 2, 3}` |
| `None` | "Ничего". Аналог Dart `null` | - | `None` |

### Мутабельные vs неизменяемые

```
  Можно изменить:           Нельзя изменить:
  ┌──────────────────┐      ┌──────────────────────────────────┐
  │ list, dict, set  │      │ int, float, str, tuple, frozenset│
  │                  │      │                                  │
  │ my_list[0] = "x" │      │ my_str[0] = "H" → TypeError!   │
  │ → ок             │      │                                  │
  └──────────────────┘      └──────────────────────────────────┘
```

## Операторы и циклы

| Оператор | Что делает | В Dart |
|----------|-----------|--------|
| `//` | Целочисленное деление: `7 // 2` → `3` | `~/` |
| `**` | Степень: `2 ** 10` → `1024` | `pow(2, 10)` |
| `%` | Остаток: `7 % 3` → `1` | `%` |
| `and`, `or`, `not` | Логические операторы | `&&`, `\|\|`, `!` |
| `is` | Identity (один объект?) | `identical()` |
| `in` | Принадлежность: `x in list` | `list.contains(x)` |
| `i += 1` | Инкремент | `i++` (в Python `++` нет!) |

| Цикл | Python | Dart |
|------|--------|------|
| По коллекции | `for item in items:` | `for (var item in items)` |
| По индексу | `for i in range(n):` | `for (var i = 0; i < n; i++)` |
| Пока | `while cond:` | `while (cond) { }` |
| Пропустить | `continue` | `continue` |
| Выйти | `break` | `break` |
| После цикла | `for...else` | Нет аналога |

## Функции

| Термин | Что это | Пример |
|--------|---------|--------|
| `*args` | "Лишние" позиционные → `tuple` | `def fn(a, *args):` → `fn(1, 2, 3)` → `args = (2, 3)` |
| `**kwargs` | "Лишние" именованные → `dict` | `fn(name="Neo")` → `kwargs = {"name": "Neo"}` |
| `lambda` | Анонимная функция, одна строка | `lambda x: x * 2` |
| Замыкание | Функция запоминает переменные из окружения | Как в Dart |
| Scope (LEGB) | Порядок поиска переменных | Local → Enclosing → Global → Built-in |
| `global x` | Изменить глобальную из функции | Нет в Dart |
| `nonlocal x` | Изменить переменную из внешней функции | Нет в Dart |

---

## ООП

| Термин | Что это |
|--------|---------|
| `self` | `this` из Dart. Но пишется явно в каждом методе |
| `__init__` | Инициализатор (не конструктор!). Настраивает объект |
| `@property` | Метод → атрибут. Без скобок: `user.full_name` |
| `@staticmethod` | Метод без `self`. Обычная функция в классе |
| `@classmethod` | Получает класс (`cls`). Для альтернативных конструкторов |
| Наследование | `class Child(Parent):` + `super()` |
| Dunder-методы | `__str__`, `__repr__`, `__eq__` — Python вызывает сам |
| ABC | Абстрактный класс: `class Base(ABC):` + `@abstractmethod` |
| Итератор | Объект с `__iter__()` + `__next__()`. `for` использует под капотом |

## pytest

| Термин | Что это |
|--------|---------|
| Фикстура | Готовит данные/объекты. Pytest подставляет по имени аргумента |
| `scope` | Время жизни: `function` (каждый тест) / `session` (все тесты) |
| `conftest.py` | Файл с фикстурами. Pytest находит автоматически |
| Параметризация | `@pytest.mark.parametrize("id", [1,2,3])` → 3 прогона |
| Маркер | `@pytest.mark.api` → `pytest -m api` |
| `assert` | Проверка. Упал → тест красный |

# ЧАСТЬ 2. ПОДВОХ-ВОПРОСЫ

> То, что реально спрашивают. Каждый вопрос — ловушка.

---

## Типы и переменные

---

### `(!)` Что будет: `a = [1,2,3]; b = a; b.append(4); print(a)`?

**Ответ:** `[1, 2, 3, 4]`

`b = a` не копирует — обе бирки на одном списке.
Чтобы скопировать: `b = a.copy()` или `b = a[:]`

---

### `(!)` Что будет: `"hello" + 5`?

**Ответ:** `TypeError`

Python не конвертирует молча. Надо: `"hello" + str(5)` или `f"hello{5}"`.
В JS сработало бы (`"hello5"`), в Python — нет.

---

### `(!)` Почему `0.1 + 0.2 != 0.3`?

**Ответ:** IEEE 754 — дробные хранятся приближённо в двоичной системе. Это во **всех языках**, не баг Python. Для точных вычислений — модуль `decimal`.

---

### `(!)` `[] == False`?

**Ответ:** `False`!

Пустой список **falsy** (`bool([]) == False`), но `[] == False` — это сравнение объектов.
Правильная проверка: `if not my_list:`

---

### `(!)` Разница `is` и `==`?

**Ответ:**
- `==` — сравнивает **значения**
- `is` — это **один и тот же объект** в памяти?

```python
a = [1]; b = [1]
a == b  # True  — значения одинаковые
a is b  # False — разные объекты
```

Для `None` всегда `is None`.

## Мутабельность

---

### `(!)` Что будет с дефолтным аргументом-списком?

```python
def add(item, items=[]):
    items.append(item)
    return items

add("a")  # → ["a"]
add("b")  # → ?
```

**Ответ:** `["a", "b"]`

Дефолтный список создаётся **ОДИН РАЗ**. Все вызовы шарят один объект.
Правильно: `items=None` → `if items is None: items = []`

---

### `(!)` Tuple нельзя изменить, но `t = ([1,2],); t[0].append(3)` работает?

**Ответ:** tuple хранит **ссылки**. Ссылка на список не меняется (tuple неизменён), но сам список мутабельный — его содержимое можно менять.

---

### `(!)` `a = "hello"; a[0] = "H"` — что будет?

**Ответ:** `TypeError`. Строки неизменяемые. Новая строка: `a = "H" + a[1:]`

---

## Функции

---

### `(!)` Что вернёт функция без `return`?

**Ответ:** `None`. Всегда. Ошибки не будет — молча вернёт `None`. Коварно при дебаге.

---

### `(!)` Разница `def fn(a, b)` и `def fn(a, *, b)`?

**Ответ:** `*` заставляет `b` передавать **только по имени**: `fn(1, b=2)`.
Без `*` можно позиционно: `fn(1, 2)`.

---

### `(!)` `lambda` может содержать `if`?

**Ответ:** только тернарный: `lambda x: "yes" if x > 0 else "no"`.
Обычный `if:` с блоком — нет. Нужна логика → обычная функция.

## ООП

---

### `(!)` Что будет если забыть `self`?

```python
class Dog:
    def bark():        # забыл self!
        print("Woof")

Dog().bark()
```

**Ответ:** `TypeError: bark() takes 0 positional arguments but 1 was given`.
Python передаёт объект первым аргументом, а функция не ждёт ничего.

---

### `(!)` `__init__` — это конструктор?

**Ответ:** технически нет. `__init__` — **инициализатор** (настраивает объект).
Конструктор — `__new__` (создаёт объект). Но все говорят "конструктор" и имеют в виду `__init__`.

---

### `(!)` Есть ли настоящие приватные атрибуты?

**Ответ:** нет.
- `_name` — конвенция "не трогай", доступ не заблокирован
- `__name` — name mangling, Python переименовывает в `_ClassName__name`:

```python
class User:
    def __init__(self):
        self.__secret = "hidden"

u = User()
# u.__secret          → AttributeError
u._User__secret       # → "hidden" — достать можно!
```

Python доверяет разработчику.

## Коллекции

---

### `(!)` `dict.get("key")` vs `dict["key"]`?

**Ответ:**
- `dict["key"]` → `KeyError` если нет ключа
- `dict.get("key")` → `None` (или дефолт: `.get("key", 0)`)

Если не уверен что ключ есть → `.get()`

---

### `(!)` `{}` — dict или set?

**Ответ:** `dict`! Пустой set → `set()`.

```
{1, 2}    → set
{"a": 1}  → dict
{}        → dict (!)
```

---

### `(!)` Что быстрее — `x in list` или `x in set`?

**Ответ:** `set` — **O(1)**, `list` — **O(n)**. Set использует хеш-таблицу. Часто ищешь → используй set.

---

### `(!)` `list.sort()` vs `sorted(list)`?

**Ответ:**
- `.sort()` — меняет **на месте**, возвращает `None`
- `sorted()` — возвращает **новый** список, оригинал не трогает

---

### `(!)` Что такое `pop()`?

**Ответ:** удаляет и **возвращает** элемент.
- `list.pop()` — последний
- `list.pop(0)` — первый
- `dict.pop("key")` — по ключу

Нет элемента → `IndexError` / `KeyError`

## Общие вопросы

---

### `(!)` Python — интерпретируемый или компилируемый?

**Ответ:** и то, и другое. Сначала компилируется в байткод (`.pyc`), потом CPython выполняет его. Для нас — интерпретируемый: пишешь → запускаешь.

---

### `(!)` Python — строго или слабо типизированный?

**Ответ:** **строго**. `"5" + 3` → `TypeError`. Не конвертирует молча (JS: `"53"`).
Но **динамически** типизированный — тип переменной можно менять на лету.

---

### `(!)` Что такое GIL?

**Ответ:** Global Interpreter Lock — замок, не даёт потокам выполнять Python-код одновременно.
- `threading` не ускоряет CPU-задачи
- Для параллелизма → `multiprocessing` или `asyncio`
- Python 3.13+ — экспериментальный режим без GIL (`--disable-gil`, PEP 703)
- Для тестов GIL не мешает — bottleneck в сети и браузере

---

### `(!)` `copy` vs `deepcopy`?

**Ответ:**
- `copy` — поверхностная (контейнер скопирован, вложенные объекты — те же)
- `deepcopy` — всё рекурсивно

```python
a = [[1]]
b = copy(a)
b[0].append(2)  # a тоже изменится!  → [[1, 2]]

c = deepcopy(a)
c[0].append(3)  # a не изменится     → [[1, 2]]
```
